# SKILL.md — Visual Essay 制作スキルパッケージ

> 本ファイルは `essays/` ツールの**技術仕様**です。新しいコラム記事を作るとき、`essays/CLAUDE.md` を読んだあとに必ず本ファイルへ目を通してください。書式・部品・ナビゲーションの最新標準は、ここで定義します。

---

## 1. 設計の三本柱

Visual Essay は「商業誌の長文記事を、単一HTMLでスマホでも読める」品質に仕上げる前提で作ります。柱は三つです。

1. **中央寄せの単一カラム** — 920px のチャプターを画面中央に配置し、行幅・図・タイトルを共通の縦軸に揃える
2. **ナビゲーション三点セット** — ヒーロー直後の「目次」セクション + デスクトップの固定サイドレール + モバイルのフローティングドロワー
3. **0章プレリュード(任意)** — 専門的な題材では、本論の前に「そもそも〜とは」を解く0章を置く

これらは [`examples/meme-coin-and-value.html`](examples/meme-coin-and-value.html) を凍結リファレンスとして参照します。迷ったら、まずそのファイルを読みます。

---

## 2. レイアウト固定値(変更禁止)

`templates/base.html` のデザイントークンと、本ファイルで定義する以下の固定値は、**指示がない限り一切変えません**。配色だけは [`templates/palettes.md`](../../templates/palettes.md) から選んで差し替えます。

### 2-1. 主要寸法

| 要素 | 最大幅 | 配置 |
|---|---|---|
| `.chapter` | **920px** | `margin: 0 auto`(中央寄せ) |
| `.ch-body` | 780px | `margin: 0 auto`(章内で中央) |
| `.ch-lede` | 780px | `margin: 0 auto 4rem`、`text-align: center` |
| `.dgm`(図表) | 920px | `margin: 3rem auto` |
| `.toc-section` | 1080px | `margin: 0 auto` |
| `.refs` | 820px | `margin: 6rem auto 2rem` |

### 2-2. 中央寄せルール

- **ヘッダ系**(ch-kicker / ch-title / ch-subtitle / ch-lede)はすべて中央揃え
- **本文段落**(`.ch-body p`)は読みやすさのために左揃えのまま、ただしカラム自体は中央
- **図表ブロック**(.dgm / .timeline / .tier)も中央寄せ

詳しくは [components.md](components.md) のレイアウト基本骨格を参照。

### 2-3. タイポグラフィ(再掲・変更禁止)

`essays/CLAUDE.md` §6-2 の通り、フォント構成・本文サイズ・行間・字間は固定です。本スキルはそれを前提に組み立てています。

## 3. ナビゲーション三点セット(必須)

新規記事は、原則として以下の三つを**全て**実装します。

### 3-1. ヒーロー直後の「目次」セクション

ヒーロー(タイトル + リード + scroll cue)の真下に、章リストを2列カードで掲示します。

- 6章なら 2×3、4章なら 2×2、3章なら 3×1 など、章数に応じて grid を調整
- 各カード: 番号 (Fraunces italic) + 章タイトル + サブタイトル + 矢印
- ホバー: 上に浮く + 左にゴールドのバー + 矢印が右に伸びる
- クリックで該当章 (`#ch0`〜`#chN`) へスクロール

HTML/CSS の雛形は [components.md §1](components.md) にあります。

### 3-2. デスクトップ固定サイドレール(`.rail`)

ビューポート幅 ≥1180px で表示される、画面左の縦型インジケータです。

- 各章を直径28px のドット(番号入り)で表示し、縦線で連結
- スクロールで現在地のドットがゴールドに点灯・拡大(IntersectionObserver で自動追従)
- ホバーで章タイトルがツールチップ風に出現
- クリックで該当章へジャンプ

### 3-3. モバイルのフローティングメニュー + ドロワー

ビューポート幅 <1180px で、画面右下に丸いハンバーガーボタンを表示します。

- タップで半透明オーバーレイ + 下から滑り上がるドロワーが展開
- ドロワー内に章リスト(番号・タイトル・サブタイトル・矢印)
- 背景タップ・ESC キー・項目選択でクローズ
- 現在章はゴールドでハイライト

### 3-4. JavaScript

スクロール位置から現在章を判定し、レール/ドロワーの両方に `.active` クラスを付ける IntersectionObserver を用意します。`rootMargin` の値は `examples/meme-coin-and-value.html` の `script` ブロックから流用してください。

## 4. 章構成の決め方

### 4-1. 章数の標準

`essays/CLAUDE.md` §3 の表を継承します。ただし、**専門的な題材で前提知識が必要な場合は、0章プレリュードを追加**してよい運用に拡張します。

| 題材の性質 | 推奨構成 | TOC レイアウト |
|---|---|---|
| 単一概念の深掘り | 3章 + 導入 + 結び | 3×1 |
| 歴史・比較もの | 5章 | 2×2 + 1(中央) or 2×3 |
| 多層・学際 | 7章 | 2×4 |
| 専門題材で前提解説が必要 | **0章 + 5章 = 6章** | 2×3 |

### 4-2. 0章プレリュードの設計原則

0章は「専門用語に飛び込む前の安全地帯」です。守るべきは三つ。

1. **比喩を3〜4個に絞り、反復する**
   暗号資産記事なら「帳簿・鍵・ブロック・指紋」の四つで一貫させる。途中で別の比喩を持ち込まない。

2. **トーンは中立・誠実な技術解説**
   本論がネガティブ寄りでも、0章は**フラットに技術を説明**する。ここで偏ると、本論の批判が説得力を失う。

3. **末尾に橋渡しの段落を置く**
   章の最後に「ここまでが技術。次章以降では、この上に乗ったものを扱う」と明示し、トーン転換に備える。

### 4-3. 図(ポンチ図)の標準量

0章には **3〜4枚の図**を入れます。比喩の各単位に一枚ずつ対応させる構造が読み手に親切です。図の作り方は §5 と [components.md §2](components.md) を参照。

## 5. ポンチ図(`.dgm`)の作法

抽象概念や仕組みを SVG で図解するときの標準コンポーネントが `.dgm` です。

### 5-1. 構造

```
.dgm
├── .dgm-head
│   ├── .dgm-kicker  (例: "Diagram 01 / 帳簿の二つのモデル")
│   └── .dgm-title   (例: "中央集権モデルと、分散モデル")
├── .dgm-svg-wrap
│   └── <svg viewBox="0 0 W H">...</svg>
└── .dgm-caption     (図の解釈を1〜3文で。strong で要点を太字に)
```

完全なコードは [components.md §2-1](components.md) にあります。

### 5-2. SVG の作法

- **viewBox 必須**。レスポンシブにスケールするため `width="100%" height="auto"`(CSS で `dgm-svg-wrap svg` に指定済み)
- **配色は CSS 変数経由で**。SVG 内の `fill="#1a1d22"` 直書きは可だが、ハイライト色はパレット由来の値を使う
- **テキストには専用クラス**を使う:
  - `.svg-label` — 主要ラベル(13px 太字)
  - `.svg-sub` — 補助説明(11px ミュート色)
  - `.svg-en` — 英訳サブ(Fraunces italic 11px)
- **矢印は `<defs><marker>` で再利用**。ハードコードしない
- **左右比較の二パネル構成**は viewBox 880×360 を基準に左右で分割
- **長尺の連鎖**(ブロックチェーンの鎖など)は viewBox 900×280 で横長に
- **チャートを描く場合**は y 軸目盛・グリッド線・ピーク注釈・X軸ラベルまで揃える

### 5-3. ポンチ図のトーン

- **子供っぽいイラスト調にしない**。アイコン化された図形と、整った字間で硬質に
- 装飾より構造を優先。三本柱の比較なら三カラム、時系列なら横軸、原因→結果なら矢印
- 色は accent-primary / accent-secondary / mute / line / warn の5階調以内に抑える

## 8. 既知の落とし穴

### 8-1. SVG のフォント指定

SVG 内の `<text>` で CSS 変数(`var(--f-jp-display)`)を使うと、ブラウザによって描画されない場合があります。**SVG では CSS 変数ではなくフォント名を直書き**してください。

```svg
<!-- OK -->
<text font-family="'Zen Old Mincho', serif" ...>
<text font-family="Fraunces, serif" font-style="italic" ...>

<!-- NG -->
<text font-family="var(--f-jp-display)" ...>
```

### 8-2. ch-body の max-width と図のはみ出し

`.ch-body` は 780px、`.dgm` は 920px の最大幅を持ちます。**`.dgm` を `.ch-body` の中に入れると 780px に切り詰められます**。広く使いたい図は ch-body の外に出すか、`.dgm` の max-width を 780px に揃えてください(現状は ch-body の中で 780px に収まる前提で運用)。

### 8-3. IntersectionObserver の閾値

`rootMargin: '-20% 0px -45% 0px'` は「ビューポート上端から20%下がった位置〜下端から45%上の位置」を判定領域にする設定です。0章が短いと判定が外れる場合があるので、章ごとに最低 800px 以上の高さを確保してください。

### 8-4. ドロワー展開時の本文スクロール固定

ドロワーが開いたとき、`document.body.style.overflow = 'hidden'` で本文スクロールを止めます。閉じる際に必ず空文字に戻すこと。戻し忘れるとページがフリーズします。

### 8-5. 二重スクロールバー

`html` と `body` の両方に `overflow` を設定すると、Safari でスクロールバーが二重になります。`body` 側だけで管理してください。
