# SKILL.md — daisu-column 制作スキルパッケージ

> 本ファイルは `column/` ツールの **数学コラム専用スキル**(daisu-column)の技術仕様です。`column/CLAUDE.md` で本スキルが選ばれたとき、必ず本ファイルを読みます。
>
> **位置づけ**: 受験数学と大学数学の橋渡しを目的とする硬派な数学読み物。著者(両大学合格経験を持つ)が、受験で出会った題材を大学数学の道具・定理の特殊例として読み直す、という核コンセプトを共有します。
>
> **トーンの参照源**: 東京出版『大学への数学』および大数ゼミ補助プリント(`column/source/` に格納)を書きっぷりの見本として参照します。ただし column 全体のトーン規約([`../../CLAUDE.md`](../../CLAUDE.md) §7)は遵守 — キャッチコピー的な見出しや AI 臭メタファーは禁止のままです。
>
> **凍結リファレンス**: [`examples/dimension-and-equations.html`](examples/dimension-and-equations.html)(第 1 回「空間と図形の次元」)。レイアウト・図方式・KaTeX 設定で迷ったら、まずこのファイルを読みます。

## 0. 設計の三本柱

1. **受験の入口・大学の本体**: すべての記事は、受験で誰もが触れた具体的な手筋・問題を入口にし、その背後にある大学数学の構造を本体として展開します。入口を省略しない。読者が高 2〜高 3 だからです
2. **必要十分の明示主義**: 大数ゼミノート流に、定義・条件は必ず必要条件と十分条件の両方を意識して書きます。「sup と max の違い」を高校生に伝えるような書き方を継承します
3. **本音併走の解説**: 「思いつくのは現実的ではありません。理系なら素直に微分のほうが確実」のような、解法の優劣を率直に評価する一文を恐れません

## 1. 文体ハイブリッドの規約

column 全体のアカデミックトーン(`column/CLAUDE.md` §7)を底に置きながら、**本文の道具命名・メタ言及には大数の格言性を入れる**ハイブリッドで書きます。

### 1-1. 採用する大数の書きっぷり(本文に入れてよい)

| 技法 | 例 | 入れる場所 |
|---|---|---|
| 「いつ / どのように」二項提示 | 「いつ:和を積で評価するとき / どのように:積一定になるように使う」 | 道具紹介の冒頭 |
| 必要十分の二点分解 | 「最低点が 20 点 ⇔ 全員 20 点以上 ∧ ちょうど 20 点の人がいる」 | 定義導入時 |
| 本音の優劣評価 | 「現実的ではありません。理系なら素直に微分」 | 別解の比較 |
| メタ脇注(`<p class="meta-note">`) | 「タイトルは 2 次関数ですが、2 次方程式・2 次不等式もここで扱います」 | 章導入直後の小フォント注 |
| ハンドルの短い命名 | 「次数下げならシュワルツ」「シュワルツはルートの二乗」 | 定理・手筋の見出し直下のサブタイトル位置 |

### 1-2. 採用しない大数語(column のトーン規約に抵触)

- ✗ 「**秘策**」「**裏技**」 — 標語調・宣伝口調なので使わない(`column/CLAUDE.md` §7-2 / §7-6 抵触)
- ✗ 「**ズバリ**」「**バッチリ**」 — 口語俗語なので使わない
- ✗ 章タイトルでの格言調 — 章タイトルは内容ベースで抑制的に書く(後述 §2)
- ✗ 「**わけです**」「**ということです**」「**となります**」 — column §7-2 禁止表現。直接断定で言い換える

### 1-3. 数学コラム特有のトーン追加ルール

- **専門用語は初出で 1 行かみ砕く**(column 共通ルールの応用)。ただし、かみ砕いた後も**正式な定義はきちんと書く** — レベルを落とさない
- **「実は」を恐れない**。受験では暗記項目だった式が、大学では定理の系として導出される瞬間を、「実は」一語で繋いでよい(ただし 1 記事 3〜5 回まで)
- **数式の前後は文章で十分につなぐ**。「次の式が成り立ちます」だけで式を投げない。「次の式は、〜が〜であることを意味します」のように、式の意味論を文で添える

## 2. 章タイトルと見出しの書き方

column 全体の §7-6(キャッチコピー禁止・標語禁止)を遵守します。**大数調の格言性は本文・小見出しレベルに留め、章タイトルは抑制的に書く**のが本スキルのハイブリッド方針です。

### 2-1. 章タイトル(`.ch-title`)
- **抽象度の高い名詞句**(または事実の短い陳述)。例: 「平方完成」「対角化の影」「内積空間としての受験数学」
- 体言止め標語(「〜を。」「〜に。」)は**禁止**
- 比喩動詞(「立ち上がる」「紐解く」)は**禁止**

### 2-2. 章サブタイトル(`.ch-subtitle`、英訳)
- 章タイトルの英訳。学術風(例: "Diagonalization in Disguise")
- 派手な英語コピーは避ける

### 2-3. 章リード(`.ch-lede`、90 字以内)
- 「この章で何を扱うか」の事実陳述。比喩や問いかけで始めない
- 例: 「平方完成は、対称行列の対角化を 2 次形式に限定した特殊例として理解できます。本章では具体例から始めて、その対応を確認します。」

### 2-4. 道具命名や手筋ラベル(本文中の小見出し `<h3>` 相当)
- ここは大数調の格言性を入れてよい
- 例: 「いつ / どのように」「次数下げならシュワルツ」「ルートの二乗」
- ただし、章タイトルや見出しでは使わない

## 3. 章構成テンプレート

数学コラム単発記事の標準構成は次の通り。題材によって章数を 5〜7 で調整します。

```
ヒーロー
  - タイトル(抑制的)
  - リード文 3 行(全体像 → 受験との接続 → 本記事の狙い)
  - スクロール誘導

01. 受験での顔(入口)
02. 受験での解き直し(中継)
03. 大学からの読み返し(本体)
04. 入試問題で確かめる(検証)
05. 余韻(結び)
```

5 章で足りる題材なら 03 と 04 を統合、7 章必要なら 03 を「定義」「定理」「証明スケッチ」に分けて拡張します。

## 4. 図表現方式 — 凡例方式・stack 方式・タイムライン方式

> **本セクションは本スキルの最も重要な技術指針です。** 第 1 回コラム制作時に「SVG 内 KaTeX」「foreignObject + KaTeX」「HTML overlay over SVG」を順に試して全部失敗した経緯があり、これらの試行錯誤を経て確立した**唯一動く 3 方式**だけを使います。新しい方式を発明しようとしないこと。

### 4-0. 図表現の絶対禁止事項(これをやると確実に失敗する)

| 禁止 | 理由 |
|---|---|
| **SVG `<text>` 内に `$...$` や `\(...\)` を書く** | KaTeX auto-render が誤って処理し、文字が消える / 位置が崩れる |
| **SVG `<text>` 内に Unicode subscript/superscript (`₀ ₁ ²`)** | Fraunces/Georgia などのフォントが対応せず、`o` や `2` など「普通の文字の小さい版」に化ける |
| **`<foreignObject>` + KaTeX を SVG 内に埋め込む** | KaTeX レンダリング後のサイズと foreignObject 座標が噛み合わず、ラベル同士が確実に重なる |
| **SVG 内 `<text>` ラベル + HTML overlay の混在** | overlay の % 座標と SVG viewBox のズレでラベルがずれる |
| **複雑な数式を SVG 内に書く**(例: \(x_0\,x + y_0\,y = r^2\)) | 上記すべての問題が複合して破綻する |
| **番号バッジ(丸+数字)** | 図中の点と紛らわしい(「丸=点なのか?」と読者が混乱) |

### 4-1. 凡例方式(`.dgm-content`)— 1 つの図 + 数式説明の標準形

**用途**: 1 つの図に複数の要素(点・ベクトル・図形)があり、それぞれを数式で説明したいとき。**最も多用する方式**。

**構造**:
- 左に **SVG 図形のみ**(背景・軸・点・線・矢印・ポリゴン)
- 右に **HTML 凡例**(KaTeX 数式 + 日本語説明)
- 図の中には**短い記号ラベル**(P, n, ∇F, A, u, v など)だけを SVG `<text>` で書く
- 凡例で「記号 → 数式・意味」の対応を表示

**コピペテンプレ**: [`components.md`](components.md) §3 を参照

**典型例**:
- Diagram 02 (2D 直線、方程式表示 / パラメータ表示)
- Diagram 03 (3D 平面)
- Diagram 06 (階数定理 V → W)
- Diagram 07 (陰関数+勾配、円と球面)
- Diagram 08 (双対空間)
- Diagram 09 (東大 2017 正六角形)
- Diagram 10 (和歌山県立医大 半球面)

### 4-2. stack 方式(`.dgm-stack` + `.dgm-stack-row`)— 段階を追う図

**用途**: 同じ題材を **条件を変えて複数の段階で示す** とき(段階が 3〜4 個)。

**構造**:
- 各段階を 1 つの `.dgm-stack-row`(横並び 1 列の grid: 左に小さな SVG、右に短いテキスト)
- 縦に複数行を積む
- テキストは「ステージタグ + 条件 + 短い解説」の 3 要素

**コピペテンプレ**: [`components.md`](components.md) §4 を参照

**典型例**:
- Diagram 01 (パラメータと次元の 4 段階)
- Diagram 04 (式の本数 → 次元の 3 段階)
- Diagram 05 (接空間ズーム 3 ステージ)

### 4-3. タイムライン方式(`.tl-list`)— 純 HTML

**用途**: 時系列の歴史的イベントを並べる。

**構造**: SVG をやめて純 HTML+CSS。縦タイムラインで、各イベントを `.tl-item`(左ボーダー色 + 左に丸ドット + 年・人名・説明)。

**コピペテンプレ**: [`components.md`](components.md) §5 を参照

**典型例**:
- Diagram 11 (次元論の一世紀: Cantor 1877 → Peano 1890 → Brouwer 1911 → Hausdorff 1918 → Grothendieck 1960s)

### 4-4. 図のラベル — 「記号ラベル方式」が標準

**図の中に書いてよいもの**(SVG `<text class="fig-lbl">` で):
- 単一英字記号: `P`, `Q`, `R`, `A`, `B`, `n`, `p`, `u`, `v`, `f`, `V`, `W`
- 短いギリシア文字: `φ`, `θ` (Unicode で OK)
- 数学記号: `∇F` (Unicode の `∇` で OK)
- 短い関数名: `ker f`, `im f` (Unicode で OK)
- 短い日本語ラベル: `接平面`, `接線`(`fig-lbl-jp` クラス、12〜13px)
- 軸ラベル: `x`, `y`, `z` (`fig-axis` クラス、italic serif)

**図の中に書いてはいけないもの**:
- KaTeX 数式 (`\(...\)`, `$...$`)
- 添字付き変数 (`x_0`, `y_0`) — これらは凡例側に
- 複雑な式 (`ax + by + cz = d`) — これらは凡例の `legend-foot` または `legend-eq` に
- 番号 (`①, ②, ③`)
- 丸の中の数字バッジ

**凡例側の対応**(`legend-item.sym`):
- `<span class="legend-sym">P</span>` で小さな色付きボックスに記号を表示
- 隣に「点 \((x_0, y_0, z_0)\)」のような KaTeX 数式付き説明
- 凡例タイトル → 主要な記号 → 補助的な記号 → 凡例フッター(主要式)の順で並べる

## 5. 数式表記

### 5-1. KaTeX 設定

- **KaTeX v0.16 系、CDN 経由**(jsdelivr)
- **`ignoredTags` に必ず `'svg'` を含める** — SVG 内のテキストを KaTeX が誤って処理しないようにするための保険

```js
renderMathInElement(document.body, {
  delimiters: [
    {left: '$$', right: '$$', display: true},
    {left: '\\[', right: '\\]', display: true},
    {left: '$',  right: '$',  display: false},
    {left: '\\(', right: '\\)', display: false}
  ],
  throwOnError: false,
  ignoredTags: ['script', 'noscript', 'style', 'textarea', 'pre', 'code', 'option', 'svg']
});
```

完全な `<head>` 組み込みは [`components.md`](components.md) §1 を参照。

### 5-2. 数式の書き場所

| 場所 | 方式 |
|---|---|
| 本文 `<p>` 中のインライン | `\(...\)` または `$...$` |
| 本文中のディスプレイ式 | `<div class="math-display">$$...$$</div>` |
| 番号付きディスプレイ式 | `<div class="math-numbered">` + `<div class="math-display">` + `<div class="math-num">(★)</div>` |
| 図の凡例(`.legend-item`) | KaTeX 数式 `\(x_0\)` などをそのまま書ける |
| 凡例フッター(`.legend-foot.eq`) | KaTeX 数式 OK |
| **SVG 内** | **KaTeX 構文 `\(...\)` を絶対書かない**(§4-0) |

### 5-3. 番号付き式

重要な式には番号を振ります。`(★)` `(♭)` のような大数調の表記、または `(1)` `(2)`。

## 6. 数学的正確性チェックリスト(過去レビューで指摘された致命級)

執筆時に以下を**必ず**確認します(audit-review でも漏れた場合は致命級扱い)。

### 6-1. ベクトル方程式の表現範囲

- [ ] `OP = sOA + tOB` で `s, t ∈ ℝ` とすると、これは **平面全体** を覆う(三角形の内部ではない)
- [ ] 三角形内部は `s, t ≥ 0` かつ `s+t ≤ 1` が必要
- [ ] 線分 AB は `s + t = 1, 0 ≤ s, t ≤ 1`
- [ ] 図と本文は同じ条件で揃える(三角形の図を描くなら本文も三角形条件で書く)

### 6-2. 等式 vs 不等式の役割

- [ ] **等式** を 1 本足すと次元が 1 下がる(線形独立かつ矛盾しない場合)
- [ ] **不等式** は次元を保ったまま範囲を切り詰めるだけ(次元は下げない)
- [ ] 「条件をもう一本足すごとに次元が下がる」は不正確 — 等式と不等式を区別

### 6-3. 直線・平面の方程式

- [ ] 直線標準形 \(\frac{x-x_0}{p} = \frac{y-y_0}{q} = \frac{z-z_0}{r}\) は \(p, q, r \neq 0\) のときに限る
- [ ] 0 を含む場合は別表現(成分ごとの式)が必要

### 6-4. 「方程式の本数 = 次元の低下」格言

- [ ] 引用するなら **「独立かつ矛盾しないとき」** の条件付け補足を1段落入れる
- [ ] 反例: \(x = 0\) と \(2x = 0\) は重複(独立でない)、\(x = 0\) と \(x = 1\) は矛盾(空集合)

### 6-5. 線形写像 vs アフィン写像

- [ ] \(f(t) = \vec{x_0} + t\vec{p}\) は **アフィン写像**(線形写像ではない)
- [ ] 階数・退化次数定理は線形部分 \(L(t) = t\vec{p}\) に対して述べる
- [ ] 「線形写像(に定数を足したもの)」より「アフィン写像」と呼ぶ方が後の論述と整合

### 6-6. 双対空間と内積

- [ ] \(V\) と \(V^*\) は同型だが、**自然に同一視されるのは標準内積を介して**
- [ ] 受験で「法線ベクトル \((a, b, c)\)」と呼ぶものは、本来 \(V^*\) の住人
- [ ] 線形汎関数 \(\phi(\vec{x}) = ax + by + cz\) と \((a, b, c) \in \mathbb{R}^3\) は標準内積で対応

### 6-7. 滑らかな多様体としての次元

- [ ] 「拡大すると \(n\) 次元の開球に見える」は **滑らかな多様体の局所モデル** の定義
- [ ] 一般の集合に対する次元論(ハウスドルフ次元、被覆次元)はこれとは別物
- [ ] 本文で「滑らかな曲線・曲面に限る」と限定するか、明示的に注記する

### 6-8. 代数幾何の余次元

- [ ] 「\(k\) 本の独立な多項式 → 余次元 \(k\)」は条件付き(完全交叉などが必要)
- [ ] 反例: \(x^2 + y^2 = 0\) は 1 本の式だが実点は原点だけ
- [ ] 「典型的に」「完全交叉のような追加条件」と弱める

### 6-9. ブラウワーの定理の名前

- [ ] 「\(\mathbb{R}^n\) と \(\mathbb{R}^m\) が同相なら \(n = m\)」は **次元不変性定理**
- [ ] **領域不変性定理** は別の主張(\(\mathbb{R}^n\) 内の開集合の同相像が再び開集合)
- [ ] 両者を混線しない。次元不変性は領域不変性から導かれる

### 6-10. ベクトル図形の幾何学的整合性

- [ ] **法線ベクトル** は対象の線・面と **直交** する向きで描く(平行ではない!)
- [ ] **方向ベクトル** は対象の線と **平行** な向きで描く
- [ ] **接ベクトル** は接平面内で接点から伸ばす
- [ ] 必ず内積で直交を計算してから SVG 座標を決める

## 7. リサーチの徹底ルール

数学コラムは題材の理解が浅いと即座に破綻します。執筆前に必ず以下を行います。

1. **一次資料の確認**: 教科書(高校・大学)、原論文、定義の確定したリファレンス(Wikipedia は二次)
2. **入試問題の収集**: 該当題材を扱った東大・京大・旧帝大の過去問を最低 3 題、可能なら 5 題以上集める
3. **大学テキストの参照**: 該当領域の標準的な大学初年次テキスト(線形代数なら齋藤・佐武・松坂、解析なら杉浦・小平・高木)から該当項目の章を読み、定理ステートメントの揺れを確認
4. **数学史の文脈**: その概念が誰によって・なぜ導入されたかの経緯を 1 段落分は調べる(章 05 の余韻で使えることが多い)
5. **関連分野への波及**: 物理・情報・統計など他分野でその道具がどう使われているかを 1〜2 例ピックアップ
6. **入試問題の出典確認**: 大学名・年度・前期/後期/文系/理系・第何問 までを正確に。**問題番号の取り違えは codex audit-review で必ず指摘されるので執筆時に確実にする**

リサーチ結果はまず記事スラグの `_research.md`(中間メモ)に書き出し、それから本文を組み立てます。
