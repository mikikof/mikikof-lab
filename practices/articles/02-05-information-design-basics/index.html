          単元の体系

          この単元は、効果的なコミュニケーションのための4つの主軸で読める。① 誰にどう届くか(コミュニケーションの分類)、② 何でわかりやすくするか(可視化・構造化・抽象化)、③ 誰でも使えるようにするか(アクセシビリティ・UD など)、④ どう見せるか(色彩)。

          直接 / 間接 × 同期 / 非同期

          コミュニケーションは「位置関係(対面か離れているか)」と「同期性(同時か時間差か)」の2軸で4象限に整理できる。手段の種類を覚えるより、組合せで捉えると応用が効く。

          直接 × 同期
          Face / Real-time
          対面 + 同時
          同じ場所にいて、双方向のやり取りが同時に進む。基本は対面の会話。
          例会話、プレゼンテーション
          間接 × 同期
          Remote / Real-time
          離隔 + 同時
          離れた相手と、双方向のやり取りを同時に行う。通信手段が必要。
          例音声チャット、オンライン会議
          直接 × 非同期
          Face / Async
          対面 + 時間差
          同じ場にいるが、やり取りに時間差がある。掲示や記録物による伝達。
          例掲示板、貼り紙(同所での非同期)
          間接 × 非同期
          Remote / Async
          離隔 + 時間差
          離れた相手と、異なる時間でやり取りを行う。物流や通信手段を用いる。
          例手紙、電子メール
          表 / 図 と LATCH の5観点

          収集したデータを視覚化するとき、媒体は表(数値・文字の配列)と図(グラフ・ダイアグラム)に大別される。さらに整理の観点は5つに集約できる。それが LATCH(Location, Alphabet, Time, Category, Hierarchy)。

          L 位置
          物理的な配置
          場所・位置関係を基準に整理する。例: ショッピングモールの各階別店舗リスト。
          A アルファベット
          五十音・言語的順序
          文字の並び順で整理する。例: クラスのあいうえお順の出席番号。
          T 時間
          前後関係
          時間軸の前後で整理する。例: 写真データを撮影日時順に並べる。
          C 分野
          類似性・関連性
          物事の種類や関連で整理する。例: 部活動を運動部と文化部に分ける。
          H 階層
          大小
          数量の大小で整理する。例: 持久走のタイムが速い順にリストを作る。
          同じデータでも観点を変えると見え方が変わる: 同じ店舗一覧を「位置」で並べると地図的に、「階層」(口コミ評価順)で並べるとランキング的になる。目的が観点を選ぶ。
          3つの構造化手法 + 5つの関係性

          情報の構造化は、収集した情報を問題解決のために利用しやすい形にまとめる作業。代表的な手法は3つ。さらに情報どうしの関係を5種類に分けると、図示の選択もしやすい。

          特性要因図
          Fishbone
          問題 ← 要因(枝)
          原因の整理

          観点を決めて問題の要因を整理・図示する。フィッシュボーンチャートとも呼ばれる。課題の発見に向く。

          KJ法
          KJ Method
          意見をカード化 → 階層化
          情報の階層化

          収集した情報を階層化し、互いの関係性を明らかにする手法。話し合いの結果整理によく使われる。

          コンセプトマップ
          Concept Map
          概念A ↔ 概念B
          概念の関係図示

          問題に含まれる概念とそれらの関係を図示する。役割や関係の可視化に向く。

          ①並列同時に存在できる関係(例: 複数のカテゴリが横に並ぶ)
          ②分岐条件によって択一的に存在する関係(例: Yes/No による進路)
          ③順列順序性がある関係(例: 工程・手順)
          ④因果原因と結果の関係
          ⑤階層上位・下位の関係(例: 組織図・概念のツリー)
          UD / バリアフリー / アクセシビリティ / ユーザビリティ / アフォーダンス / シグニファイア

          「誰でも使えるようにする」周辺の用語は混同されやすい。出発点(障壁の除去か、最初から全員か)と対象(物の存在自体か、特定の行為への誘導か)で区別する。

          ユニバーサルデザイン
          最初から全員に
          • 文化・年齢・性別・能力にかかわらず使いやすい
          • はじめから前提として組み込む

          恩恵を受ける人数で採用是非を決めない。すべてのデザインの前提。

          バリアフリー
          既存の障壁を除く
          • 制約がある人に対し、その障壁を取り除く
          • 後付けの解決策が中心

          UD と対比されるが排他的ではない。段差にスロープを足すのはバリアフリー。

          Aユーザビリティ製品やサービスを利用して目的を達成する際の使いやすさそのもの。
          Bアクセシビリティどんな条件でも、すべての人に幅広く情報へのアクセスが確保されていること。複数のメディアでの提供が要点。
          Cアフォーダンスものの存在自体が、人にさまざまな行動を想起させる作用。ドアは押す/引く/スライドのいずれもできる可能性がある。
          Dシグニファイアその想起のうち特定の行動へ誘導するためにデザインされたもの。ドアの金属プレートは「押す」を選ばせるシグニファイア。
          色相 / 明度 / 彩度 と 色相環

          色には3つの軸がある。色相は色の違い(色相環で環状に配置)、明度は明るさ(高は白、低は黒)、彩度は鮮やかさ(高は鮮明、低は無彩色)。文字の視認性は背景色との明度・彩度の差が大きいほど高い。

          三要素
          色相色の違い。色相環で環状に配置
          明度明るさ。高は白、低は黒
          彩度鮮やかさ。低は無彩色(白・グレー・黒)
          色相環の関係
          類似色色相環で隣り合う色 → まとまり感
          補色色相環で向かい合う色 → 派手・対比
          同系色同じ色相で彩度・明度が異なる色
          COLOR WHEEL — 12分割色相環
          ①〜⑫の番号で色相環上の位置を指す
          × 赤と緑(色相だけが違う組合せ)は判別しにくい人がいる。明度・彩度の差を付ける、網かけや形状など色以外の手段と組み合わせる工夫が要る。

          コミュニケーションの形態

          次の文章中の空欄に適当な語句を入れよ。

          　古くから身近にあるコミュニケーションの手段として、相手と対面しての会話があり、これは(　①　)コミュニケーションである。さらに、この形態のコミュニケーションは、双方向のやり取りが同時に行われる(　②　)型コミュニケーションでもある。
          また、同じく古くからのコミュニケーションの手段である手紙は、離れた相手と行われる(　③　)コミュニケーションであり、双方向のやり取りが異なる時間で行われる(　④　)型コミュニケーションでもある。
          語群(4語) — タップで空欄に入れる
          ✓ 正答: ① 直接 / ② 同期 / ③ 間接 / ④ 非同期
          ベストフィット
          コミュニケーションの分類は「直接か間接か」と「同期型か非同期型か」の組合せである。
          解説

          古くから身近にあるコミュニケーションの手段は、会話のような「直接かつ同期型のコミュニケーション」か、手紙のような「間接かつ非同期型のコミュニケーション」がおもなものであった。現代では、電話の普及によって「間接かつ同期型のコミュニケーション」が身近なものとなり、さらに情報技術が発達することで、さまざまなコミュニケーションの手段が使われるようになった。

          2軸で4分類になる(補足)

          位置関係(直接 / 間接)と同期性(同期 / 非同期)という独立した2軸を組み合わせると、4つの形態が一意に決まる。手段名を覚えるよりも、「会話 → 直接×同期」「手紙 → 間接×非同期」「電話 → 間接×同期」のように軸の組合せで捉えるほうが応用が利く。

          例電話、オンライン会議

          次の⑴〜⑸のデータの整理の仕方は、どのような基準に基づいているか答えよ。

          選択肢(LATCH の5観点)
          (ア)位置 (イ)アルファベット(五十音) (ウ)時間 (エ)分野 (オ)階層
          ⑴
          持久走のタイムが速い順に生徒のリストを作る。
          ⑵
          写真データを撮影日時順に整理する。
          ⑶
          学校の部活動を運動部と文化部に分ける。
          ⑷
          クラスであいうえお順に出席番号を付ける。
          ⑸

          ベストフィット
          データを整理する際の観点は、「位置」、「アルファベット(五十音)」、「時間」、「分野」、「階層」の五つである。
          解説

          データを整理する際の五つの観点は、その頭文字(Location(位置)、Alphabet(アルファベット)、Time(時間)、Category(分野)、Hierarchy(階層))を取って「LATCH(ラッチ)」と呼ばれる。同じデータを整理するとしても、目的によって違う観点を用いる場合もある。例えば、お店などを紹介するWebサイトが、口コミの評価順に並べて表示すれば「階層」で整理していることになるが、地域ごとに分けて表示すれば「位置」で整理していることになる。

          数量の大小は「階層」(補足)

          「持久走のタイムが速い順」は数量(タイム)の大小で並べる整理。LATCH の H = Hierarchy(階層) が該当する。LATCH の「階層」は数量的な大小による整理。組織図のような上位・下位関係は、後出の「情報の構造化」で扱う関係5種の「階層」として区別する。

          「ショッピングモールの各階」は物理的な配置による整理で L = Location(位置)。同じ「階」という日本語が出てくるが、観点は別なので注意。

          頭文字で覚えるデータの整理軸

          ⑸ ショッピングモール
          物理的な位置・配置で整理
          A アルファベット
          ⑷ あいうえお順
          五十音・言語的順序で整理
          T 時間
          ⑵ 撮影日時順
          時間の前後関係で整理
          C 分野
          ⑶ 運動部・文化部
          類似性・関連性で整理
          H 階層
          ⑴ 持久走の速い順
          数量の大小で整理

          情報の抽象化

          次の(ア)〜(オ)の記述のうち、情報の抽象化の手法として適当なものをすべて選べ。

          複数選択 — 該当するものすべて
          ✓ 正答: (イ)、(エ)、(オ)
          ベストフィット
          情報の抽象化とは、相手に伝える必要がある情報に絞って表現することであるため、現実の事象を省略したり単純化したりすることになる。
          解説

          ピクトグラムは、単純な図形の組合せでデザインされることが多いため、写実的な表現にはならない。また、色についても、情報を表現する要素の一つであるため、使用する色はできるだけ少なくして必要な情報のみを表現するようにしている。

          「写実」と「抽象」の対比(補足)

          (ア)(ウ) 適当でない:写実的に描く・地形を正確に再現するのは「写実化」であり、抽象化の方向とは逆。(イ)(エ)(オ)はいずれも「色を減らす」「形を単純化する」「直線的に描く」という情報を絞り込む操作で、抽象化の手法そのもの。

          地下鉄の路線図が代表例。実際の路線は曲がりくねっているが、利用者は「どの駅でどの線に乗り換えるか」さえわかればよいので、路線は直線で描かれる。これが情報の抽象化。

          ABSTRACTION — 写実 vs 抽象
          伝える目的に応じて情報を絞る
          写実的(× 抽象化ではない)
          (ア)ピクトグラムを写実的に描く
          (ウ)路線図で地形を正確に再現
          抽象的(○ 抽象化の手法)
          (イ)使用する色を減らす
          (エ)校舎の形状を単純化
          (オ)道路を直線的に描く

          ユニバーサルデザイン

          次の(ア)〜(オ)の記述のうち、ユニバーサルデザインの説明として適当なものをすべて選べ。

          複数選択 — 該当するものすべて
          ✓ 正答: (イ)、(エ)
          ベストフィット
          はじめからすべての人が使いやすくしてあるデザインがユニバーサルデザインである。
          解説

          ユニバーサルデザインの恩恵はすべての人が受けられるものなので、何かをデザインする際には、ユニバーサルデザインの採用が前提となる。障がい者や高齢者の障壁を取り除くのは、バリアフリーの考え方である。

          UD と バリアフリー の混同を避ける(補足)

          (ア) 適当でない:恩恵を受ける人数で採否を判断するのは UD の考え方ではない。UD はすべてのデザインの前提として組み込む。(ウ) 適当でない:特定の人の障壁を取り除くのはバリアフリー。(オ) 適当でない:訴えが出てから事後対応する発想も、UD ではなくバリアフリーに近い。

          (イ)(エ) はそれぞれ「対象が全員」「最初から組み込む」という UD の核心を述べている。

          UD vs BARRIER-FREE
          出発点と対象が違う
          ユニバーサルデザイン
          出発点最初から全員に

          JIS(日本産業規格)で規定されている案内用図記号(ピクトグラム)において、注意を表す図記号の配色が黄色と黒であることについて、次の(ア)〜(エ)の記述のうち、最も適当なものを一つ選べ。

          ✓解説
          正答
          (イ)
          解説

          無彩色と色相環の関係(補足)

          白・グレー・黒は無彩色で、色相環には登場しない。だから黒は他の色と補色や類似色の関係になり得ない。視認性を上げるための判断軸は、黒との場合は「明度の差」が主役になる。
