          3つの柱 — 守る・防ぐ・取り締まる

          この単元は3本の柱で構成される。①守る対象(個人情報とプライバシー)を理解し、②セキュリティの考え方(3要素・認証・有害情報対策)で防御を組み立て、最後に③関連法規(不正アクセス禁止法・特定電子メール法・情報流通PF対処法)で社会的なルールを押さえる。

          生存する個人 + 識別可能性

          個人情報保護法でいう「個人情報」とは、生存する個人に関する情報で、次のいずれかにあたるもの。①単独またはいくつかの情報を組み合わせると個人を特定できるもの、②個人識別符号(身体特徴を電子計算機用に変換した符号や、個人に割り当てられる番号)。故人の情報は対象外。

          基本四情報
          Identifiable
          単独 or 組合せ
          氏名・住所・生年月日・性別。これだけで、または組合せで個人を特定できる
          例氏名と住所、生年月日と性別の組合せ
          身体特徴の符号化
          Biometric ID
          個人識別符号 ①
          DNAの塩基配列、指紋、静脈、虹彩などの身体特徴を、電子計算機で識別できるよう変換した符号
          例指紋データ、顔認証の特徴量、虹彩データ
          割当番号
          Assigned Number
          個人識別符号 ②
          旅券番号、個人番号(マイナンバー)など、個人に割り当てられる番号
          例パスポートの旅券番号、マイナンバー
          × 個人情報ではない例:故人の氏名・住所(生存していない)、グループ名(単独では特定不可)。ただし他情報と組み合わせて個人を識別できる場合は個人情報になり得る。
          プライバシー権 / 肖像権 / パブリシティ権

          「プライバシー」と一括りにされがちだが、保護対象と根拠は3つに分かれる。プライバシー権は私生活上の事柄、肖像権は自分の姿の利用、パブリシティ権は有名人の名前・肖像がもつ経済的価値を守る。

          プライバシー権
          Privacy
          私生活 非公開
          趣味・嗜好 家庭内
          私生活を守る

          他人に知られたくない私生活上の個人的な情報を、公開・干渉・侵害から守る。法律で明記されておらず、裁判の判例で確認されている。

          肖像権
          Likeness
          顔・姿 無断撮影禁止
          無断利用禁止
          自分の姿を守る

          自分の肖像(顔・姿)を、許可なく撮影・利用されないように主張できる権利。撮影時の合意目的を超えた利用も侵害になる。

          パブリシティ権
          Publicity
          名前 肖像
          経済的価値
          経済的価値を守る

          有名人が自分の名前や肖像のもつ経済的価値(顧客吸引力)を独占できる権利。無断のグッズ化・商品化が典型的な侵害。

          機密性 / 完全性 / 可用性

          情報セキュリティは 機密性(Confidentiality)・完全性(Integrity)・可用性(Availability) の3要素で評価する。何かの事故が起きたとき、どの要素が損なわれたかを言語化することが、対策の出発点になる。

          機密性
          Confidentiality
          許可された人だけ見られる
          許可されたものだけが情報にアクセスできること。漏洩・盗難で損なわれる
          例USBメモリ盗難で顧客情報が外部流出
          完全性
          Integrity
          壊れていない・正確
          情報が破壊・改竄・消去されていないこと。改ざんやデータ削除で損なわれる
          例顧客情報が誤った内容で運用されている
          可用性
          Availability
          必要なときに使える
          必要なときに情報やシステムにアクセスできること。サイバー攻撃や障害で損なわれる
          例サイバー攻撃で取引システムが停止
          判定のコツ: 漏洩・盗難 → 機密性 / 改ざん・誤データ → 完全性 / 停止・遅延 → 可用性。1つの事故で複数要素が同時に損なわれることもある。
          不正アクセス禁止法 / 特定電子メール法 / 情報流通PF対処法

          情報を取り巻くトラブルの種類ごとに、対応する法律が用意されている。侵入を取り締まる法、迷惑メールを規制する法、SNSの誹謗中傷に対応する法、の3つを分けて押さえる。

          不正アクセス禁止法
          侵入・なりすましを禁止
          権限なくコンピュータへアクセスする行為(セキュリティホール攻撃、なりすまし、他人のID/パスワードの無断使用・無断提供など)を禁止する。
          特定電子メール法
          広告メールを規制
          大量に送信される広告宣伝メール・迷惑メールを規制する。広告宣伝メールの送信はオプトイン方式(事前承諾)が義務付けられている。
          情報流通PF対処法
          SNS等の権利侵害に対処
          SNS・掲示板等の投稿による権利侵害に対し、発信者情報開示、事業者の免責条件、大規模PF(プラットフォーマー)の削除申出対応・透明化義務などを定める。
          切り分けの要点: 「不正侵入」 → 不正アクセス禁止法 / 「勝手に広告メール」 → 特定電子メール法 / 「SNSで誹謗中傷」 → 情報流通PF対処法。「個人情報の本人同意なし第三者提供」は個人情報保護法で別法。
          オプトイン / オプトアウト / 匿名加工情報

          個人情報は本人の同意を得るか、個人を特定できないように加工することで活用できる。同意の取り方には、事前承諾を要するオプトインと、事前拒否されない限り使えるオプトアウトがある。広告宣伝メールはオプトイン方式が義務。

          ①オプトイン方式事前に「受け取りたい」と承諾した人にのみ提供する。広告宣伝メールはこの方式が法律で義務化されている(特定電子メール法)。
          ②オプトアウト方式事前に「受け取りたくない」と意思表示した人以外には提供する。配信停止の手段を必ず提示する必要がある。
          ③匿名加工情報特定の個人を識別できず、元の個人情報を復元できないよう加工した情報。一定のルール(提供項目・提供方法の公表、識別行為禁止 等)の下で本人同意なしに第三者提供できるため、データ流通の促進に使われる。
          覚え方: 「in(中に入る) = 事前承諾」「out(外に出る) = 事後拒否」。広告宣伝メールは必ずオプトイン(法律で義務化)。配信停止手段の明示はオプトアウト方式の運用条件。

          3方式の使い分け(補足)

          オプトインは「承諾した人にだけ提供」 — 事前に明示的な同意が必要な、最も慎重な方式。広告宣伝メールはこちらが法律上義務化されている。

          匿名加工情報は「個人を特定できず、復元もできないよう加工」 — 氏名・住所など識別情報を取り除き、所定の基準で加工した情報。一定のルール(提供項目・提供方法の公表、識別行為禁止 等の義務)の下で本人同意なしに第三者提供できる。データ流通促進が目的。

          プライバシーとは、趣味や嗜好など、他人が知らない個人の私生活上の事柄で、公開を望まない情報である。個人情報は、指紋やマイナンバーなど、特定の個人を識別できることが要件となる。

          情報セキュリティの3要素

          次の⑴〜⑶の事象で、情報セキュリティの3要素のうちで損なわれた要素を答えよ。

          選択肢(損なわれた要素)
          (機密性)許可された人だけが情報にアクセスできること (完全性)情報が破壊・改竄・消去されていないこと (可用性)必要なときに情報にアクセスできること
          ⑴
          取引先との電子取引システムがサイバー攻撃を受け、処理ができなくなった。
          ⑵
          顧客情報のコピーが保存されたUSBメモリが盗難にあった。
          ⑶
          顧客情報管理システムに保存された顧客情報が誤った内容のまま運用されていた。
          ✓ 正答: ⑴ 可用性 / ⑵ 機密性 / ⑶ 完全性
          ベストフィット
          情報セキュリティは、「機密性」、「完全性」、「可用性」の三つの要素からなる。
          解説

          〈パスワード〉

          数字[0-9] 4桁で作成されたパスワードを、すべての組合せを試す総当たり攻撃により、1秒で解読される不正アクセス攻撃を受けた。同じ攻撃を受けたときに総当たりが終了するまでに20分以上時間がかかるようにしたい。⑴引き続き数字[0-9]のみでパスワードを作成する場合、⑵数字[0-9]と英小文字[a-z]の組合せでパスワードを作成する場合、それぞれ何桁が必要となるか求めよ。ただし、攻撃に必要な時間は、パスワードの組合せ数のみに影響されるものとする。

          ⑴ 数字[0-9]のみで作成する場合: ①
          ⑵ 数字+英小文字で作成する場合: ②
          語群 — タップで空欄に入れる
          ✓解説
          正答
          ⑴ 8桁 / ⑵ 5桁
          解説

          数字4桁の組合せ数は 10000 通りである。20分(1200秒)以上かかるようにするには 12,000,000 通り以上にする必要がある。

          ⑴ 数字のみでは 7桁で 10,000,000 通り(これは1200万に届かない)、8桁で 100,000,000 通り(これは1200万を超える)となるため、8桁必要。

          ⑵ 数字と英小文字を足すと 36文字 となる。4桁で 1,679,616 通り(1200万に届かない)、5桁で 60,466,176 通り(1200万を超える)となるため、5桁必要。

          解読時間からの逆算手順(補足)

          本問は「解読時間 → 必要桁数」を求める典型問題。手順は3ステップ。

          ① 1秒あたりの試行数を求める:現状 4桁・10000通り を 1秒で解読 → 1秒あたり 10000 試行できる攻撃力。

          ② 目標時間に対する組合せ数を求める:20分 = 1200秒。1200秒で解読されない量 = 10000 × 1200 = 12,000,000 通り超が必要。

          ③ 文字種ごとに必要な桁数を求める:10n ≥ 12,000,000 → n=8(107=10,000,000 では足りない)。36n ≥ 12,000,000 → n=5(364=1,679,616 では足りない)。

          文字種を増やすと必要桁数が大きく減る(8桁 → 5桁)のがポイント。組合せ数の増加が指数的だから、文字種拡張の効果が桁数より大きく出る場合がある。
