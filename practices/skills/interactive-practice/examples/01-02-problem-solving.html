          情報社会と問題解決の体系

          この単元では、まず私たちが生きる情報社会の姿(AI・IoT・超スマート社会)とそこで生じる課題を押さえる。次に、課題を解決するための問題解決の手順(5ステップ)と、その中で使う把握・収集の手法(ブレスト・トレードオフ・一次/二次情報)を学ぶ。

          AI / IoT / 超スマート社会

          現代の情報社会は、AI(人間の知的活動の一部をコンピュータが担う技術)と IoT(あらゆるモノがインターネットに繋がる仕組み)が支えている。これらを基盤として、社会のさまざまなニーズにきめ細かに対応できる超スマート社会(Society 5.0)が目指されている。

          人工知能(AI)
          Artificial Intelligence
          知的活動の一部を担う
          人間の知的な活動の一部をコンピュータにさせることを目的とする技術や研究分野
          例音声アシスタント、画像認識、自動翻訳
          IoT
          Internet of Things
          モノがネットに繋がる
          身のまわりのあらゆるものがインターネットに繋がる仕組み
          例スマート家電、コネクテッドカー、ウェアラブル端末
          超スマート社会
          Society 5.0
          きめ細かに応える社会
          社会のさまざまなニーズに、AI・IoT を駆使してきめ細かに対応できる社会
          例遠隔医療、自動運転、スマートシティ
          サイバー犯罪 / 情報格差 / VDT症候群 / テクノストレス

          情報技術の発達は便利さをもたらす一方、新しい課題も生んでいる。犯罪・格差・健康・心理の4つの軸で起こる典型的な問題を押さえる。

          サイバー犯罪
          情報通信ネットワーク上の犯罪
          ネットワークを使って行われる犯罪。不正アクセス、フィッシング詐欺、ウイルス配布、なりすまし、SNS 上の誹謗中傷など。
          情報格差(デジタルデバイド)
          利用できる/できないの差
          インターネットなどの ICT を利用できる者とできない者との間で生じる格差。世代・地域・経済・障害などの要因で発生する。
          VDT症候群
          心身に生じる症状
          ディスプレイを長時間使用したことにより心身に生じる症状。眼精疲労・肩こり・腰痛・頭痛など。
          テクノストレス
          ICT への適応に関する心理
          ICT への適応が原因のストレス。テクノ不安症(適応できず不安を感じる)とテクノ依存症(過度に依存する)の2タイプがある。
          AI を支える3つのキーワード

          「AI」は広い概念。中で 機械学習 が手法、生成AI が出力の形、大規模言語モデル(LLM) が文章生成 AI の中核となる仕組み、という階層関係がある。

          機械学習
          Machine Learning
          • コンピュータがデータから自動的に学習する
          • パターンの発見、予測、判断を行う
          • AI の中核技術として性能向上を支えた

          「AI が進歩した」という表現の中身は、ほとんどの場合機械学習の精度向上を指す。深層学習(ディープラーニング)はその一形態。

          生成AI と 大規模言語モデル
          Generative AI / LLM
          • 生成AI:学習済データを活用して新たなコンテンツを作る AI(文章・画像・音声・動画・コード)
          • 大規模言語モデル(LLM):インターネット上の膨大なテキストを学習し、文章を生成できる AI
          • 文章生成系の生成AIは、多くが LLM を中核にもつ

          関係:AI ⊃ 機械学習 ⊃ 生成AI。文章を生む生成AI のエンジンとしてLLM が広く使われている。

          発見 → 明確化 → 検討 → 決定 → 実施・評価

          問題解決は、いきなり解決案を実行するのではなく、5つのステップを順に進める。前段(発見・明確化)を飛ばすと「そもそも何の問題か」がぶれて、後段で迷走する。

          5ステップの流れ
          5 Steps
          ① 問題の発見 → ② 問題の明確化 → ③ 解決案の検討
          ④ 解決案の決定 → ⑤ 実施と評価
          それぞれの目的

          ① 身のまわりのどこに問題があるかを明らかにする / ② 問題解決の目標とゴールを設定する / ③ 情報を収集・整理・分析し、複数の解決案を作る / ④ 評価して解決案を1つに決める / ⑤ 実施し、結果を評価する。

          PPDAC サイクル
          PPDAC Cycle
          Problem → Plan → Data
          Analysis → Conclusion
          統計・データ分析でよく使う

          問題(Problem)→ 計画(Plan)→ データ収集(Data)→ 分析(Analysis)→ 結論(Conclusion)を循環させる手法。情報の問題解決でも、同じ精神(順序立てて回し続ける)が共通している。

          サイクルである: 5ステップも PPDAC も、1回で終わりではなく 繰り返し回す ことで問題を継続的に改善していく考え方。PDCA サイクル(Plan-Do-Check-Action)と同じ精神。
          ブレスト4ルール / トレードオフ / 一次情報・二次情報

          問題解決の前段では、問題の把握と情報の収集が大切になる。ブレーンストーミングでアイデアを発散させ、トレードオフ関係に注意し、一次情報と二次情報を区別して集める。

          ①ブレーンストーミング(把握の手法)グループで自由に考えを出し合う。ルールは 4つ:批判禁止 / 自由奔放 / 質より量 / 便乗歓迎。質より量の段階なので、突飛なアイデアも歓迎する。
          ②トレードオフ(関係の見極め)一方を達成するために他方を犠牲にしなければならない関係。例:利便性 ↔ プライバシー、コスト ↔ 性能。解決案を比較するときに「何を犠牲にするか」を意識する。
          ③一次情報(オリジナル)自分自身が経験するなどオリジナルの情報。自分の体験、アンケート、フィールドワークでの観察など。
          ④二次情報(まとめ)すでに書籍・論文・Web ページなどにまとめられた情報。一次情報を基に作成された情報。
          ⑤フィールドワークアンケート調査や実物観察など、現場で直接情報を収集する活動。一次情報を得る代表的な方法。
          × 一次情報 = 信頼性が高い、二次情報 = 信頼性が低い、ではない。両者の違いは「自分が直接得たか / 既にまとめられたか」であって、信頼性や利用価値に直接の関係はない。

          Q3

          問題の把握(ブレーンストーミング)

          次の⑴〜⑷について、ブレーンストーミングで推奨される行動には○を、推奨されない行動には×を記せ。

          ⑴ 他人の意見に便乗した意見を言う。
          ⑵ 効果がありそうな意見に厳選して発言する。
          ⑶ 他人の意見で誤りがある場合には批判する。
          ⑷ 自由奔放に、突飛な発言をする。
          ✓ 正答: ⑴ ○ / ⑵ × / ⑶ × / ⑷ ○
          ベストフィット
          ブレーンストーミングを表す言葉=「批判禁止」「自由奔放」「質より量」「便乗歓迎」
          解説

          ブレーンストーミングでは、①他人の考えに対して批判や否定をしない。②奇抜な考えやユニークなアイデアを歓迎する。③質より量を重視する。④別々のアイデアを結合・変化させて新たなアイデアを生み出す、という4つのルールがある。

          4ルールへの当てはめ(補足)

          本問の正誤は、4ルールのどれに当てはまるかで決まる。

          ⑴ ○ 便乗歓迎:他人の意見に乗っかって新しいアイデアを出すのは、結合・変化のルールに直接合致する。発散段階では「乗っかる」が効率がよい。

          ⑵ × 質より量に反する:厳選してから発言すると、案の母数が少なくなる。発散の段階では「とりあえず数を出す」が原則で、選別は後段(④ 解決案の決定)でやる。

          ⑶ × 批判禁止に反する:誤りがあっても発散の最中に批判するとアイデアが萎む。批判は後段の検証フェーズに回す。

          ⑷ ○ 自由奔放:突飛な発言は、独創的なアイデアの種になる。「実現可能性」は後段で考える。

          批判禁止
          No criticism
          ⑶ × の根拠
          他人のアイデアを否定しない
          対応誤りでも発散段階で批判しない
          自由奔放
          Wild ideas welcome
          ⑷ ○ の根拠
          奇抜・突飛なアイデアを歓迎
          対応突飛な発言 ○
          質より量
          Quantity over quality
          ⑵ × の根拠
          数を出すことが目的
          対応厳選して発言は ×
          便乗歓迎
          Build on ideas
          ⑴ ○ の根拠
          他人のアイデアの結合・変化
          対応便乗した発言 ○

          Q4

          情報の収集(一次情報)

          次の(ア)〜(エ)のうち、一次情報に該当するものをすべて選べ。

          複数選択 — 該当するものすべて
          ✓ 正答: (ア)、(ウ)
          ベストフィット
          一次情報は、自分自身の経験や経験した人から得られるオリジナルな情報である。
          解説

          一次情報は自分自身で経験したり収集したりした情報や、経験した人から直接聞き取った情報を指す。書籍や Web サイトに掲載された情報は二次情報となる。それぞれの特性を問題解決にいかす必要がある。

          一次/二次の見分け方(補足)

          区別の基準はシンプルで、「自分(または経験した人)から直接得たか / 既にまとめられたものか」の一点に尽きる。

          (ア) アンケート調査の結果:自分が設問を立てて集めた一次情報。フィールドワークの代表例。

          (ウ) 自分の経験:自分自身がオリジナルに体験した一次情報。観察・体験記録・実験結果も同類。

          (イ) 購入した書籍 / (エ) Web サイト:いずれも他人がまとめた二次情報。元になった一次情報があり、それを編集・整理したもの。

          注意点:一次情報のほうが偉い・信頼性が高い、ではない。両者の違いは「直接性」だけで、信頼性や利用価値とは直接の関係はない。一次情報にも個人の偏見が入り得るし、二次情報にも査読を経た高信頼のものがある。

          PRIMARY vs SECONDARY — 直接性で分ける
          「自分が直接得たか」「既にまとめられたか」
          一次情報(オリジナル)
          (ア) ○アンケート調査の結果
          (ウ) ○自分の経験
          他フィールドワーク・実物観察・実験結果
          二次情報(まとめ)
          (イ) ×購入した書籍の情報
          (エ) ×Web サイトの情報
          他論文・新聞記事・百科事典
          PRACTICE 1 / 8 ○ × 判定 ベストフィット 類題7
          Q1

          〈人工知能(AI)〉

          次の⑴〜⑸の記述は、生成AI の特徴について説明したものである。適当なものには○を、適当でないものには×を記せ。

          ⑴ 生成AI は文章を作ることのみが可能で、画像やコンピュータプログラムなどのコンテンツを作成することはできない。
          ⑵ 生成AI は学習データに含まれる著作物を無断で利用し、類似のコンテンツを生成する可能性がある。
          ⑶ 生成AI が生成する情報は必ず正確で、事実関係も保証されている。
          ⑷ 生成AI を利用するには高度な技術力が必要であり、一部の企業や研究機関のみが使用することに限定されるため、情報格差が生じる可能性がある。
          ⑸ 生成AI は、差別や偏見が含まれた上であたかも正しい情報であるかのような情報を生成する場合がある。
          ✓解説
          正答
          ⑴ × / ⑵ ○ / ⑶ × / ⑷ × / ⑸ ○
          解説
          • ⑴ × 適当でない。生成AI で画像やコンピュータプログラムを作成することができる。
          • ⑶ × 適当でない。生成AI が作成した文章などの情報には誤りが含まれることがある。
          • ⑷ × 適当でない。生成AI の利用は、簡易な文章での命令(プロンプト)で可能である。

          〈問題解決案の検討(重み付け順位付け)〉

          次の表は、自転車の購入にあたって、機種 A〜C に対し、評価項目ごとに順位付けをしたものである。1位に5点、2位に3点、3位に1点の点数を与え、さらに「デザイン」に 0.5、「重量」に 0.4、「変速機」に 0.1 の重みを付けて順位を付け直したとき、機種 A〜C に1番目から3番目までの順位を付けて答えよ。

          順位の表(問題)
          Ranking Table
          評価項目 機種 A 機種 B 機種 C
          デザイン(重み 0.5)1位2位3位
          重量(重み 0.4)3位2位1位
          変速機(重み 0.1)2位3位1位
          1番目:①
          2番目:②
          3番目:③
          語群(3語) — タップで空欄に入れる
          ✓解説
          正答
          1番目:機種A / 2番目:機種C / 3番目:機種B
          解説 — 計算手順

          順位の表を点数に置き換える(1位=5点、2位=3点、3位=1点)。

          次に、それぞれの点数に重み(デザイン 0.5、重量 0.4、変速機 0.1)を掛けて足し合わせる。

          • 機種A:5×0.5 + 1×0.4 + 3×0.1 = 2.5 + 0.4 + 0.3 = 3.2点
          • 機種B:3×0.5 + 3×0.4 + 1×0.1 = 1.5 + 1.2 + 0.1 = 2.8点
          • 機種C:1×0.5 + 5×0.4 + 5×0.1 = 0.5 + 2.0 + 0.5 = 3.0点

          合計点を比較すると、機種A(3.2) > 機種C(3.0) > 機種B(2.8) となり、1番目:A、2番目:C、3番目:B となる。

          重み付けで順位が変わる(補足)

          本問のポイントは、重み付けを反映する前と後で順位が入れ替わる こと。

          重み付け前(単純合計):A = 5+1+3 = 9点 / B = 3+3+1 = 7点 / C = 1+5+5 = 11点 → 1番目 C、2番目 A、3番目 B。

          重み付け後:A = 3.2点 / B = 2.8点 / C = 3.0点 → 1番目 A、2番目 C、3番目 B。

          このように、「何を重視するか」(=重み) によって最終的な解決案の順位は変わる。問題解決の④「解決案の決定」段階では、評価軸の重み付けが意思決定を左右することを意識する必要がある。

          WEIGHTED SCORING — 重み付けの計算過程
          点数 → 重み付け → 合計 → 順位
          ① 順位 → 点数
          Convert
          1位 → 5点 2位 → 3点 3位 → 1点
          順位を点数に変換

          3段階の点数を割り当てる。差を強調したい場合は重みをつけることで調整する。

          ② 点数 × 重み
          Weight
          A: 2.5+0.4+0.3 B: 1.5+1.2+0.1 C: 0.5+2.0+0.5
          評価項目の重要度を反映

          デザインを最重視(0.5)した結果、機種A(デザイン1位)が有利になる。

          ③ 合計 → 順位
          Rank
          A: 3.2点 → 1番目 C: 3.0点 → 2番目 B: 2.8点 → 3番目
          最終的な順位

          単純合計だと C が1番目だったが、デザイン重視の重み付けで A が逆転1番目となる。
