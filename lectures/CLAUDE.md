# lectures/ — Claude Code 運用指示書(Interactive Lecture Lab)

このフォルダは、大学・高校の授業で使うインタラクティブなHTMLスライドサイトを量産する場所です。**単一HTMLファイル**で投影と復習の両方に使える教材を作ります。

ルートの `CLAUDE.md` からここに到達したら、本ファイルの指示に従って作業してください。

---

## 1. このツールの位置づけ

### ユーザー像
- **作り手**:幹生(平成国際大学 情報デザイン学部 教員)
- **投影者**:授業担当者
- **閲覧者**:受講学生(大学生、および情報Ⅰ共通テスト対策を受ける進学校の生徒)

### 成果物の特徴
- 単一 HTML ファイル(CSS・JS を全て内包、オフライン閲覧可)
- 授業中はプロジェクタ投影、授業後は生徒の自習復習に使える
- 学習ノートの項番(①②③…)と完全対応
- 最新の事例・ニュースを取り込んだ生きた教材

### 公開URL
- 授業サイトトップ:`https://mikikof.github.io/mikikof-lab/lectures/`
- 単元ページ:`https://mikikof.github.io/mikikof-lab/lectures/articles/{slug}/`

## 2. ディレクトリ構成

```
lectures/
├── CLAUDE.md                       本ファイル
├── index.html                      授業サイトトップ(単元一覧)
├── skills/
│   └── interactive-lecture/        制作スキルパッケージ
│       ├── SKILL.md                技術仕様・制作手順
│       ├── template.html           空の骨組み(コピーして使う)
│       ├── components.md           部品カタログ(13種)
│       └── examples/
│           └── 07-information-security.html   凍結リファレンス
└── articles/
    └── {番号}-{スラッグ}/
        └── index.html              各単元の完成品
```

### 新規単元の命名規則
- 番号は学習ノート or 教科書の単元番号(07, 08, 09...)
- スラッグは英語ハイフン区切り(例:`information-security`, `computer-basics`)
- 配下は必ず `index.html`(GitHub Pages で `/articles/{slug}/` として公開される)

## 3. 作業着手前に必ず読むもの

新しい単元を作る・既存を修正するときは、**以下の順で必ず読んでから**コードに触る:

1. 本ファイル(このCLAUDE.md)— 制作哲学とチェックリスト
2. `skills/interactive-lecture/SKILL.md` — 技術仕様と落とし穴
3. `skills/interactive-lecture/components.md` — 部品カタログ
4. `skills/interactive-lecture/examples/` の最新完成品 — 生きた参照

これを読まずに書き始めると、毎回デザイン規約が揺れて量産の意味がなくなる。**skill を読むのは念のためではなく必須**。

## 4. 制作哲学(これを外したらやり直し)

### 4.1 答えは必ず後出し
- 穴埋め、クイズ、実習問題の正解を先に見せない
- クリック・タップで展開する仕組みに統一する
- 「?」「クリックで確認」などのプレースホルダ表示を使う
- これを破ると**教材ではなく単なる解説資料になる**

### 4.2 問題は手応えのある難度
- 「使い回しはNG」「共用PCはNG」といった自明の問題は出さない
- 引っかけ、グレー判断、複数要素の組み合わせで判断を要する問題にする
- 対象は**共通テスト〜難関大受験生レベル**
- 「これなら教科書見れば分かる」問題は価値がない

### 4.3 実例を入れる
- 抽象的な説明だけでなく、現実の事件・ニュースで具体化する
- 情報セキュリティ系は **web_search で最新事例を確認**してから書く
- 事例は展開式で格納し、スライドを情報過多にしない

### 4.4 子供っぽい表現を避ける
- ❌「やってみよう」「頑張ろう」「みんなで考えてみよう」
- ❌ 過度な絵文字(💡 など機能的記号を除く)
- ✅ 中立的・プロフェッショナルな表現

### 4.5 対象属性の明示をしない
- 「東大受験生向け」等の特定属性明示はしない
- 「このサイトで扱う内容」「共通テストで問われる論点」のような内容ベース記述にする

### 4.6 学習ノートとの対応を明示
- 各スライドのラベルに「学習ノート ①」「実習 2」など対応箇所を表示
- 項番まで揃えることで、学生が手元のノートと照合しながら進められる

## 5. 技術的な制約

### 必須
- **単一HTMLファイル**で完結
- CSS・JS は全て内包
- Google Fonts の読み込みのみ外部 CDN として許可
- Safari(macOS / iPad)と Chrome で動作

### 禁止
- ❌ フレームワーク(React, Vue, Alpine.js 等)の導入
- ❌ localStorage / sessionStorage の使用
- ❌ 重いライブラリの CDN 読み込み
- ❌ ビルドステップが必要な構成(そのまま GitHub Pages で配信できない)

### ナビゲーション(標準装備)
- 矢印キー(← →)、スペース、PageUp/Down
- タッチスワイプ(iPad 投影対応)
- F キーで全画面切替
- 進捗バー + ページ番号常時表示

詳細仕様は `skills/interactive-lecture/SKILL.md` を参照。

## 6. 標準ワークフロー(新規単元1本)

新しい授業単元の依頼を受けたら、以下の順で進める:

### [1] 素材の確認
- ユーザーから学習ノートの画像 or PDF を受領
- 項番・実習問題・POINT を洗い出す
- 対象単元番号・スラッグ・授業のコマ数を確認

### [2] 構成案の提示と合意
- カバー範囲、スライド数、インタラクション要素を案として出す
- ユーザーに選択肢で確認(迷ったら聞く)
- **合意前にコードを書き始めない**

### [3] 事例・最新情報のリサーチ(該当する場合)
- web_search で該当トピックの最新事件を確認
- 3〜5件の事例候補をピックアップして組み込む

### [4] 実装
- `skills/interactive-lecture/template.html` をコピーして `articles/{番号}-{スラッグ}/index.html` に配置
- `components.md` から必要パーツをコピーして中身を組む
- デザイントークン(`SKILL.md` 参照)は絶対に変えない

### [5] 構文・動作チェック
- 中括弧バランス、scriptタグ、divタグの対応
- 既知の落とし穴(`SKILL.md` セクション8)に照らしてチェック

### [6] ユーザーへの報告
- 変更サマリを提示
- 「次はここが気になるかも」を先回りで伝える
- 投影してのフィードバック待ち

### [7] 完成後の凍結
- 完成版を `skills/interactive-lecture/examples/{番号}-{スラッグ}.html` にもコピー
- `lectures/index.html`(単元一覧)にエントリを追加

## 7. 納品前チェックリスト

完成と判断する前に、以下を全てクリアすること:

- [ ] 学習ノートの全項番 + 全実習が網羅されている
- [ ] 答えは全てクリック/タップで後出し展開
- [ ] 紺・青系のデザイントークンが守られている(色のブレがない)
- [ ] セクション仕切りが画面中央に表示される
- [ ] 上下バーとコンテンツが衝突していない
- [ ] 矢印キー・スペース・タッチ・F キー全て動作
- [ ] 進捗バー・ページ番号が正しい
- [ ] 幅 1100px 以下でレイアウトが崩れない
- [ ] 禁止表現(「やってみよう」「東大受験生」等)が残っていない
- [ ] 事例セクションがある場合、最新(過去1〜3年)の事件が含まれている
- [ ] `examples/` に凍結コピーを配置
- [ ] `lectures/index.html` にエントリ追加

## 8. Git 運用

コミットプレフィックスは `lectures:` を使う(ルート CLAUDE.md の規約に準拠):

```
lectures: 07-information-security を新規追加
lectures: パスワード管理の選択肢を難化
lectures: SKILL.md に落とし穴を追記
```

他ツール(`essays/`)と混在させない。

## 9. 困ったとき

- 実装で迷ったら `examples/` の最新完成品を読む
- デザインで迷ったら `SKILL.md` の Design Tokens セクション
- 挙動バグに遭遇したら `SKILL.md` 末尾の「既知の落とし穴」
- 学習ノートとの対応で迷ったらユーザーに聞く(勝手に省略しない)
- それでも判断つかない場合は作業を止めてユーザーに確認
